Software development process would never be complete without a versioning system in place, Source code being written in various languages across framework/platform by multiple developers and the pain part comes in the way of versioning it.Software being distributed in public need to be versioned for better tracking and maintenance.a proper versioning system helps to scale the project with transient development team.

What is Git ?

Git, A distributed source control versioning system used today by most of the development teams to develop high quality software maintaining multiple version with teams across globe working on different time zone and as being said distributed a repository will be available in each developer machine which would be synced (Push/Pull)

Why Git ?

  • Git is a proven solution to maintain code base
  • Git is efficient in its core to maintain projects at any magnitude
  • Integrated natively/thru extension in top-notch IDEs
    • Android Studio
    • Visual Studio
    • XCode
  • Supported across OS ( Mac, Linux, Windows)
  •  Continuous Integration

Understand Git

  • Git maintains a .git folder to keep track of the changes (diff/patch) alone instead of holding the entire file
  • Change set is the snapshot of the code base at any point of time
  • Git tracks file based on the changes made against the HEAD version
  • HEAD in git is the latest available committed change set
  • Master is a trunk wherever your code changes are tracked by default
  • Supports branches for non-linear development
  • .gitconfig - maintains configuration of single git repository or globally across your system
  • .gitignore - helps to ignore the files which might be specific to IDE/User/Common library
  • Supports staging/un-staging of files

When Git ?

Git is an anonymous solution when u are considering of maintaining your source code, it could be a simple class work, college project or a successful application in the market.
